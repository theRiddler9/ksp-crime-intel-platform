# Hackathon Demo Script — Karnataka SCRB Crime Intel Platform

## Demo Scenario: End-to-End Operational Workflow (Stages 1 through 8)

### Step 1: FIR Intake at Station (Stage 1 & 2)
1. **Action:** Submit new FIR payload via `functions/intake-incident` endpoint.
   ```json
   {
     "fir_number": "FIR_BEN_HAL_2026_0999",
     "case_type": "Theft",
     "ipc_bns_sections": "IPC 379 / BNS 303",
     "district_id": "BENGALURU_CITY",
     "station_id": "STATION_HAL",
     "latitude": 12.9605,
     "longitude": 77.6485,
     "occurrence_time": "2026-07-22T21:00:00Z",
     "mo_tags": ["night time theft", "two wheeler theft"],
     "suspect_name": "Ramesh Kumar",
     "prior_record_flag": true,
     "victim_name": "Karthik N"
   }
   ```
2. **Observe:** Intake validates mandatory fields, passes duplicate check, writes to relational Data Store, and auto-transmits data upward to District & SCRB.

### Step 2: Auto-Enrichment & Graph Linking (Stage 3)
1. **Observe:** Signal `DataStore.incidents.onInsert` triggers `crosslink-on-insert`.
2. **Result:** Entity nodes and edges (`suspect->location`, `suspect->victim`) auto-populated into NoSQL graph store. MO similarity matcher identifies prior matching cases (`FIR_BEN_HAL_2026_0102`).

### Step 3: Automated Pattern & Hotspot Surfacing (Stage 4)
1. **Observe:** `flag-detector` event function evaluates spatial clusters and 30-day baseline trend spikes.
2. **Result:** New `HOTSPOT` flag generated and pushed to pending queue with confidence score `0.87`.

### Step 4: Role-Based Dashboard View (Stage 5)
1. **Action:** Request payload from `functions/role-view` with different role headers:
   - `role=Constable&station_id=STATION_HAL`: Shows local beat flags, nearby repeat offenders, patrol route overlay.
   - `role=SHO&station_id=STATION_HAL`: Shows station backlog summary, local hotspots, pending flags.
   - `role=SP&district=BENGALURU_CITY`: Shows district trend lines, station comparative stats, escalated anomaly flags.
   - `role=DGP`: Shows plain-language brief, top 5 emerging concerns, statewide KPIs.

### Step 5: Human Officer Review & Escalation (Stage 6)
1. **Action:** Post review decision to `functions/review-decision`:
   ```json
   {
     "flag_id": "FLAG_HS_1001",
     "reviewer_id": "SHO_HAL_01",
     "reviewer_role": "SHO",
     "decision": "ACKNOWLEDGE",
     "action_type": "PATROL_DEPLOYMENT",
     "assigned_to": "BEAT_CONSTABLE_04",
     "reason": "Verified cluster near HAL main road; night patrol frequency doubled."
   }
   ```
2. **Observe:** Catalyst Circuit `review-escalation-flow` executes branch `Branch_Acknowledge_GroundAction`.

### Step 6: Executive Plain-Language Synthesis (Stage 5 & 8)
1. **Action:** Call `functions/dgp-summary`.
2. **Observe:** Returns Markdown executive intelligence brief generated by QuickML RAG engine.
